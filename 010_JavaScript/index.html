Theoretical Part

  • Inheritance in ES6+ syntax, keywords extends, super
  • Creating getters and setters, keywords get, set
  • Additional JavaScript embedding capabilities in the page
  • Recommendations for using the Module pattern for simulating namespaces in JavaScript
  • Non-intrusive JavaScript
  • Concept of browser objects in client-side JavaScript
  • Basic methods for accessing HTML elements via the document object, and changing element styles
  • Concept of DOM – Document Object Model, some properties of document

Practical Part

Develop JavaScript functions to solve the following tasks. The scripts should be placed on the page using non-intrusive JavaScript style (a module created in the window.onload event handler), with processing triggered by button clicks (input elements of type button, without using forms).

Display the results of the script in the markup, use styling, flex, and navigation between pages. Each task should be placed on its own page, and the home page should contain the task description.

When first loading the page, use document.write(), and for outputs triggered by event handlers, select a specific markup element without using document.write().

Task 1. Develop a class hierarchy in ES6+ syntax to represent 3D shapes - sphere, cone, cylinder, cube. Develop methods to calculate the surface area, volume, display data about the shape, and compare volumes of 3D shapes. Create getters and setters with validation of the assigned values. Replace invalid data with default values, no error messages are needed, exceptions should not be used yet.

Also display images of the 3D shapes.

Create an array of objects of these classes – 2 objects of each type. By commands assigned to buttons, sort a copy of the array by decreasing volume, increasing surface area, and display the original array of 3D shapes.

Task 2. Design a class in ES6+ syntax to represent weather data: temperature, pressure, humidity, wind speed and direction, graphical representation of atmospheric phenomena (clear, cloudy, rain, etc. – no more than 5). Define a method to form a string for displaying data in the markup. Override the toString() method for simple console output. Create an array of weather data for the week, display it on the page. By button commands, display weather data ordered (only during output, the order of elements in the original array should not change): by decreasing temperature, increasing pressure, by decreasing wind speed. Highlight the windiest and calmest days. There should also be a button to display the original array without highlighting the elements.
